## Dependency Notes

- 001 is marked as recommended before 004 because the repo has many documented
  conventions but no `AGENTS.md`; a guide improves execution quality for all
  follow-on agent work.
- 004 must land before 005 and 006 because lazy-loading and CSP changes can
  break UI routing/rendering in ways `tsc` alone will not catch.
- 002 and 003 are independent and can run in parallel with 001.
- 007 is mostly backend/Rust persistence work and can run independently of the
  UI track.
- 008 is intentionally a design/spike plan, not a build plan.
- 009, 010, 011 form the "async commands" group from the 2026-06-25 blocking-call
  audit. All three apply the **same** mechanical pattern (make the command
  `async fn`, change `State<T>` → `State<'_, T>`) that the codebase already uses
  for `watch_repo`/`launch_difftool`. They are mutually independent and touch only
  `src-tauri/src/lib.rs`, so run them in priority order (009 first — those are the
  real multi-second UI freezes) but do NOT parallelize them in separate worktrees,
  since all three edit the same file and would conflict on merge.
